# Critic Rubric — Intake (Stage 1)

Applies to: `intake.product_brief`, `intake.interview`, `intake.research_kickoff`

## Product Brief Checks

| # | Check | PASS criteria | REVISE trigger |
|---|-------|--------------|----------------|
| 1 | App name + URL | Named + URL present (can be placeholder if idea-stage) | Completely missing |
| 2 | One-liner | What it does in one clear sentence | >2 sentences or absent |
| 3 | Target user | Specific role/persona named | "Everyone" or too vague |
| 4 | Current state | One of: idea / MVP / fully built / billing integrated | Missing |
| 5 | Revenue model | One of: free / freemium / paid / subscription / one-time | Missing |
| 6 | Max's angle | Brand fit + audience connection noted | Missing |

## Interview + Research Kickoff Checks

| # | Check | PASS criteria | REVISE trigger |
|---|-------|--------------|----------------|
| 7 | Interview documented | Q&A artifact exists and linked | Missing |
| 8 | Research kickoff brief | Seeds + questions + target sources listed | Missing or too sparse to act on |
| 9 | Deep research tasks | ≥3 specific research questions passed to Validate | None specified |

## REVISE output required
List each failing check + specific fix instruction.
