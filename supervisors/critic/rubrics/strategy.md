# Critic Rubric — Strategy (Stage 4)

Applies to: `strategy.ship_plan`, `strategy.budget`, `strategy.targets`

## Ship Plan Checks

| # | Check | PASS criteria | REVISE trigger |
|---|-------|--------------|----------------|
| 1 | Positioning one-liner | Single clear sentence using VoC language | Generic/invented language OR missing |
| 2 | Value proposition | Pain → Solution → Outcome using ICP's own words | Not traceable to VoC bank |
| 3 | Competitive positioning | ≥2 competitors with "why us over X" statements (specific, not vague) | Vague or missing |
| 4 | Pricing tiers | Named tiers + price points + included features | Missing or incomplete |
| 5 | Pricing research cited | ≥2 competitors' pricing documented | No pricing research |
| 6 | Channel strategy | ≥3 channels listed with ICP-fit rationale | Channel list without rationale |
| 7 | Segment lock | `primary_segment`, `deferred_segment`, `excluded_segments` all defined | Any segment lock field missing |
| 8 | Budget | Per-stage cost estimates + total | Missing budget |
| 9 | Success targets | Measurable targets (signups, revenue, etc.) defined per phase | Vague "grow" type targets |
| 10 | Winning angles | ≥3 hook angles for first creative batch | Missing angles |
| 11 | 30-day experiment map | `hypothesis → metric → success/fail boundary → decision window → owner` for ≥3 experiments | Missing or incomplete |
| 12 | ICP trace | Positioning + offer references specific ICP segments from validate.icp | Positioning doesn't map to ICP |

## REVISE output required
List each failing check + specific fix instruction.
