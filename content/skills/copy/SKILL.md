# Content Copy

Generate copy that sounds like a human founder, adapted per platform. All copy goes through a voice system — never write and ship raw AI output.

## Process

### 1. Clarify the Brief
Before writing, establish:
- **Topic** — core message or claim
- **Format** — platform/format (reel script, caption, thread, LinkedIn, email, blog)
- **Source material** — research, transcript, or existing content to repurpose
- **CTA goal** — what the audience should do (comment, subscribe, click, buy)

### 2. Ground in Evidence
Pull from available research:
- Audience pain language (VoC bank)
- Proven hook formulas for the format
- Competing content and what's performing

### 3. Write Using Voice Constraints

**Hook structure (persona-led):** `Persona + Pain + Outcome + Mechanism + Proof + CTA`

Voice rules:
- Use real language from pain research, not abstract claims
- Avoid AI-cadence phrases: "delve", "unlock the power of", "comprehensive solution", "game-changer"
- Match platform register (casual for reels, professional for LinkedIn, conversational for email)
- One dominant CTA per piece — never split attention

### 4. Humanize Pass
Final pass: remove AI patterns, punch up the hook, verify platform formatting.

## Platform Formats

| Platform | Format rules |
|----------|-------------|
| Instagram Reel | Hook (0-3s), problem (3-10s), solution (10-45s), CTA (last 3s). Max 150-word caption. |
| Instagram Carousel | Slide 1 = hook/promise. Slides 2-8 = one point each. Last slide = CTA. |
| Twitter/X Thread | Tweet 1 = bold hook claim. Tweets 2-8 = proof/steps. Last tweet = CTA + restate hook. |
| LinkedIn | Professional angle, metrics-led, longer form acceptable. Story → lesson → CTA. |
| Email/Newsletter | Subject line ≤50 chars. Hook in first sentence. Plain text preferred. One CTA. |
| Blog | SEO-structured H2/H3. Intro hooks with pain. One internal link per 500 words. |

## Required Outputs

- Full copy draft (format-appropriate)
- Hook variants (2-3 alternates for testing)
- CTA variants (3) by temperature: cold / warm / hot
- Platform-specific formatting applied
