# Content Page

Build page-level assets with problem-first structure and deployment-ready outputs.

## Types

- Landing pages (cold traffic → email capture / trial)
- Pricing pages (feature comparison, toggle, tier CTAs)
- Competitor comparison pages
- FAQ / objection handling pages
- Thank-you and post-signup pages

## Inputs Required

- Product brief and ICP
- Offer strategy (headline, value prop, guarantee)
- VoC quotes for social proof
- Channel intent (cold social / search / referral)

## Required Outputs

- Page copy by section (hero, proof strip, value props, objections, CTA)
- Conversion intent map (primary + secondary CTA)
- SEO essentials (title, meta description, schema notes)
- Mobile-first layout notes and asset slots
- UTM and event tracking requirements

## GTM Rules

- **Problem-first headline** — pain before feature claims
- **Hero uses VoC language** — ≥1 near-verbatim pain phrase in H1 or sub-headline
- **One dominant CTA per page** — no competing conversion actions
- **Objection handling visible above final CTA** — don't bury it in FAQ
- **Copy humanized** — no AI cadence; reads like a founder wrote it

## Page Section Template

```
HERO
  H1: [Pain phrase from VoC]
  Sub: [Outcome promise + mechanism]
  CTA: [Single action button]
  Social proof strip: [logos / review count / metric]

PROBLEM FRAME
  "If you're [persona], you probably [pain]..."
  3 bullets of specific pain evidence

VALUE PROPS (3-4)
  Icon + H3 + 2 sentences each
  Anchor to ICP pain, not generic features

PROOF
  Testimonial (verbatim, attributed)
  Metric or case study

OBJECTION HANDLING
  3-5 most common objections with direct responses
  (Source from ICP research — not invented)

FINAL CTA
  Restate outcome promise
  Button + micro-copy (risk reversal)

FOOTER
  FAQ (3-5 questions)
```

## Done Criteria

- [ ] Hero uses VoC language
- [ ] Single dominant CTA
- [ ] Objection handling section present
- [ ] Copy humanized (no AI phrases)
- [ ] Mobile layout notes included
- [ ] UTM + event tracking specified
