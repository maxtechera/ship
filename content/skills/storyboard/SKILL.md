# Content Storyboard

Plan content before creating it. Structure ideas into actionable production briefs.

## Content Package Architecture

Every content idea starts as a source piece and multiplies across platforms:

```
Source Idea
  │
  ├── Content Brief (topic, angle, CTA, target audience)
  │
  ├── Source Piece
  │     ├── Script (hook/problem/solution/CTA with timestamps)
  │     ├── Storyboard (visual plan per section)
  │     └── Recording brief (camera setup, slides needed, b-roll list)
  │
  └── Content Package
        ├── Instagram reel
        ├── Instagram carousel (5-10 slides)
        ├── Twitter/X thread (4-6 tweets)
        ├── LinkedIn post
        ├── Newsletter block
        └── Blog post (if SEO opportunity)
```

## Storyboard Format (Shot-by-Shot)

```
SECTION: Hook (0-3s)
  Visual: [what's on screen — face / text overlay / b-roll]
  Script: "[exact words]"
  Camera: [straight to camera / show screen / gesture]
  Text overlay: "[text if any]"
  B-roll: [none / b-roll slot ID]

SECTION: Problem (3-10s)
  ...

SECTION: Solution (10-45s)
  ...

SECTION: CTA (last 3s)
  ...
```

## Recording Brief

Before any recording session, create:

1. **Script** with timestamp markers and camera directions
2. **Shot list** — what camera setups are needed
3. **B-roll list** — which screen recordings or visuals to prepare first
4. **Slide deck** — any text overlays or presentation visuals
5. **Props/setup** — anything that needs to be physically ready

## Weekly Plan Format

```
Mon [Date] — Batch 1
  Record: [topics / scripts]
  Edit: [previous batch]
  Publish: [scheduled pieces]

Thu [Date] — Batch 2
  ...
```

## Done Criteria

- [ ] Shot-by-shot storyboard with visual descriptions
- [ ] Script with timestamp markers and camera directions
- [ ] B-roll slots identified for each section
- [ ] Recording brief complete (setup, props, slides)
- [ ] Content package map (source → derivatives)
